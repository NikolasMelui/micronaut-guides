7 Google App Engine

We are going to deploy the Micronaut application developed in this guide to the Google App Engine Flexible Environment

App Engine allows developers to focus on doing what they do best: writing code. Based on Google Compute Engine, the App Engine flexible environment automatically scales your app up and down while balancing the load. Microservices, authorization, SQL and NoSQL databases, traffic splitting, logging, versioning, security scanning, and content delivery networks are all supported natively.

Run the command:

gcloud app create

to initialize an App Engine application within the current Google Cloud project.

You will need to choose the region where you want your App Engine Application located.

7.1 Google App Engine Gradle Plugin

To deploy to App Engine, we are going to use the Google App Engine Gradle Plugin.

Add the plugin as a buildscript dependency:

build.gradle
buildscript {
    repositories {
        mavenCentral()
    }

    dependencies {
        classpath 'com.google.cloud.tools:appengine-gradle-plugin:2.0.0-rc4'
    }
}

Apply the plugin and specify a deployment version and the project ID. When you created the Project in the Google Cloud Dashboard you got the Project ID. Notice I didnâ€™t use dots on the version number.

build.gradle
apply plugin: 'com.google.cloud.tools.appengine'

version "0-2"
group "example.micronaut"

ext {
    appEngineProjectVersion = project.version
    appEngineProjectId = 'calm-history-228704'
}

Then configure the deployment properties:

build.gradle
appengine {
  stage.artifact = "${buildDir}/libs/${project.name}-${project.version}-all.jar"
  deploy {
      projectId = appEngineProjectId
      version = appEngineProjectVersion
  }
}

7.2 Application Deployment Configuration

To deploy to Google App Engine, we need to add the file src/main/appengine/app.yaml

We are going to configure the Java 8 runtime in the Flexible environment

The Java 8 runtime does not include any web-serving framework. The only requirement is that your app should listen and respond on port 8080. The sample code here shows how you can add your own framework, such as Spring Boot, to the Java 8 runtime.

It describes the applicationâ€™s deployment configuration:

src/main/appengine/app.yaml
runtime: java
env: flex
